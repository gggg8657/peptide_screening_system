## 1. 논문의 핵심 주장 (한 문장)

> **멀티에이전트 AI 시스템으로 SSTR2 방사성의약품 후보 탐색 전체 워크플로우를 자동화할 수 있으며, 에이전트 간 역할 분화와 협업이 단일 파이프라인 대비 품질·확장성에서 우위를 가진다.**

---

## 2. 연구 동기 및 배경

### 왜 이 연구를 하는가

- SSTR2 타겟 방사성의약품(^177Lu-DOTATATE 등)의 한계: 제한적 종양 침투, off-target 부작용
- 신규 후보 탐색은 **광범위한 화학 공간 탐색 + 복수 계산 도구 수동 조율**이 필요 → 시간·비용 과다
- 최근 AI 도구들(RFdiffusion, DiffDock, ESMFold 등)이 성숙했으나, **통합 자동화 파이프라인은 부재**

### 왜 "멀티에이전트"인가

- 단일 AI 모델로는 "구조 예측 → 도킹 → 에너지 평가 → 선택성 검증" 전 과정을 커버할 수 없음
- **역할 분화된 전문 에이전트**가 각 단계를 담당하고, 자기 개선 루프로 반복 최적화
- 개발 과정 자체에도 멀티 AI 에이전트(Cursor + Codex CLI + Claude Code)를 활용 → "에이전트가 에이전트를 만드는" 구조

---

## 3. 제안 시스템 개요

### 3.1 Dual-Silo 파이프라인

```
Silo A: 3-Arm Virtual Screening          Silo B: 제약 기반 돌연변이 생성
┌──────────────────────────┐             ┌─────────────────────────┐
│ Arm 1: 소분자 (MolMIM)    │             │ SST-14 템플릿            │
│ Arm 2: 펩타이드 변이체     │             │ → Constraint Compiler    │
│ Arm 3: De Novo            │             │ → MutantGenerator        │
│   (RFdiff→MPNN→ESMFold)  │             │ → 3-Gate HIL Filter      │
└───────────┬──────────────┘             └───────────┬─────────────┘
            └──────────┬─────────────────────────────┘
                       ▼
              통합 랭킹 → Top 후보 → 웨트랩 제안
```

### 3.2 6-Agent Hybrid 아키텍처 (핵심 기여 1)

| 에이전트 | 역할 | LLM/Code | 근거 |
|----------|------|:--------:|------|
| Planner | 실험 계획, 가설 생성 | **LLM** | 과학적 추론 필요 |
| Builder | 7-Step 파이프라인 실행 | Code | API 호출, 결정론적 |
| QC&Ranker | 4단계 품질 게이트 | Code | 수치 비교, 알고리즘 |
| DiversityMgr | 구조 클러스터링 | Code | 서열 유사도 계산 |
| Critic | 실패 분석, 개선 제안 | **LLM** | 근본 원인 추론 필요 |
| Reporter | 보고서 생성 | **LLM** | 자연어 합성 필요 |

→ LLM 3개 + Code 3개 = **LLM 호출 50% 절감**, 관심사 분리 유지

### 3.3 Competing Hypotheses 패턴 (핵심 기여 2)

아키텍처 결정의 객관성 확보를 위해 **4명의 AI 에이전트가 경쟁적으로 분석**:

- Advocate-6Agent (6-Agent 옹호)
- Advocate-Hybrid (4-Agent 옹호)
- LLM-Evaluator (모델 평가)
- Devil's Advocate (약점 공격)

→ **3:1 합의로 6-Agent Hybrid 채택**, Devil's Advocate가 가장 치명적 버그(P0) 발견

### 3.4 멀티 MCP 개발 방법론 (핵심 기여 3)

```
Cursor IDE (주 오케스트레이터)
├── Codex CLI (비판적 코드 감사)
└── Claude Code (자율 추론, 아키텍처 검증)
```

→ 3종 AI 에이전트가 **동일 코드베이스를 서로 다른 관점**으로 분석  
→ 치명적 버그 3건(P0) + 주요 버그 3건(P1) 발견·수정

## 4. 주요 결과

### 4.1 에이전트 협업으로 발견된 결함

| 심각도 | 내용 | 발견 에이전트 |

- **Silo A**: 소분자 40개 + 펩타이드 13개 + De Novo 16개 → 통합 랭킹 20개

- **LLM 호출**: 반복당 6회 → 3회 (**50% 절감**)
