## 연구 배경 및 동기

- **신경내분비종양(NETs)** 은 SSTR2를 과발현 → 방사성의약품 표적
- 기존 DOTATATE/DOTATOC는 임상 성공, 그러나 **화학적 다양성 한계**
- 펩타이드 후보 스크리닝은 본질적으로 **반복적 워크플로우**:

```
생성(mutation) → 평가(docking) → QC → 보고(reporting) → 계획(planning) → ...
```

### 핵심 질문

> SST-14 기반 돌연변이 탐색을 **자동화된 다중 에이전트 시스템**으로
> 운영하고, 그 결과를 **정량적으로 검증**할 수 있는가?

## 기존 한계점

| 항목 | 수동 프로세스의 문제 |
|------|---------------------|
| **추적성** | 어떤 돌연변이를 왜 선택했는지 기록 부재 |
| **재현성** | 연구자별 파라미터 차이, 결과 편차 |
| **편향** | 특정 위치에 돌연변이 집중 (mutation-position bias) |
| **중복** | 이전 반복에서 시도한 서열 재생성 |
| **통합** | 생성-평가-QC-보고가 분리된 도구로 수행 |

### 기존 도구의 한계

- 개별 도구(PyRosetta, FoldMason 등)는 존재하나, **전체 루프를 통합하는 시스템 부재**
- 반복 간 피드백 전달 메커니즘 없음

## 시스템 아키텍처: 5-Agent 구조

```
┌─────────────────────────────────────────────────┐
│            Agentic AI Co-Scientist               │
│              (Qwen3:8b 기반)                     │
├─────────┬─────────┬───────────┬────────┬────────┤
│ Planner │QCRanker │ Diversity │ Critic │Reporter│
│         │         │  Manager  │        │        │
└────┬────┴────┬────┴─────┬─────┴───┬────┴───┬────┘
     │         │          │         │        │
 전략수립  QC+순위매김  중복억제  실패분석  구조화로그
 돌연변이   규칙기반    편향완화  개선제안  PyMOL/CSV
  계획     게이트판정   FoldMason  패턴   Markdown
```

| Agent | 역할 | 주요 도구 |
|-------|------|-----------|
| **Planner** | 이전 반복 피드백 기반 돌연변이 전략 수립 | LLM reasoning |
| **QCRanker** | 규칙 기반 QC 게이트 + 순위 매김 | PyRosetta FlexPepDock |
| **DiversityManager** | 중복 서열 억제, 돌연변이 위치 편향 완화 | FoldMason |
| **Critic** | 실패 패턴 분석, 개선 포인트 도출 | LLM analysis |
| **Reporter** | 구조화된 로그 및 시각화 생성 | PyMOL, CSV, Markdown |

## 반복 워크플로우 (Iterative Loop)

```
  ┌─────────────────────────────────────────────────────────┐
  │                                                         │
  ▼                                                         │
Planner ──→ Mutate ──→ Dock ──→ QC Gate ──→ Critique ──→ Report
  │            │         │         │            │           │
  │         서열생성   FlexPep   ΔG>0제외     실패패턴     CSV/MD
  │                    Dock     규칙판정      분석        PyMOL
  │                                                         │
  └──────────── 피드백 (다음 반복 전략에 반영) ◄─────────────┘
```

### 각 단계 핵심 사항

- **Mutate**: SST-14 기반 서열에서 위치별 아미노산 치환
- **Dock**: 후보당 **10회 독립 도킹 시행**, 상위 3회 평균 ΔG 산출
- **QC Gate**: ΔG > 0 → refinement 실패로 판정, **즉시 제외**
- **Critique**: 탈락 후보의 실패 원인 분석 → 다음 반복 전략에 반영
- **Report**: 모든 결과를 구조화된 형식으로 기록 (추적성 확보)

## 시뮬레이션 파이프라인 상세

### FlexPepDock Refinement Protocol

```
입력: SSTR2-펩타이드 복합체 구조
  │
  ├─ PyRosetta FlexPepDock refinement
  │    └─ 유연 펩타이드 도킹 (backbone + side-chain 최적화)
  │
  ├─ InterfaceAnalyzerMover
  │    └─ 결합 에너지 (ΔG, REU) 산출
  │
  └─ 통계 처리
       ├─ 후보당 10회 독립 시행
       ├─ Top-3 평균 ΔG → 대표 점수
       └─ 표준편차(σ) → 재현성 지표
```

### 핵심 메트릭

| 메트릭 | 의미 | 판정 기준 |
|--------|------|-----------|
| Mean ΔG (top-3) | 결합 친화도 대리 지표 | 낮을수록 유리 |
| dΔG vs WT | 야생형 대비 변화량 | 음수 = WT보다 우수 |
| σ (표준편차) | 도킹 포즈 재현성 | 낮을수록 안정적 |
| QC pass/fail | ΔG > 0 여부 | fail = 제외 |

## 검증 전략: Reference Set 구성

### 3개 클래스로 구성된 검증 세트

```
Reference Set
  │
  ├─ LIT (문헌 기반)         ← 알려진 결합체와 비교
  │    ├─ LIT-01: WT SST-14 (기준선)
  │    ├─ LIT-02: Octreotide (임상 약물)
  │    └─ LIT-03: CST-14    (구조 유사 펩타이드)
  │
  ├─ SAN (Sanity Check)      ← 결합부위 잔기 치환 → 약화 확인
  │    ├─ SAN-01: W8A mutant (Trp8 → Ala)
  │    └─ SAN-02: K9A mutant (Lys9 → Ala)
  │
  └─ NOV (Novel)             ← 파이프라인 설계 신규 후보
       ├─ NOV-01: 3 돌연변이 (비핫스팟 위치)
       └─ NOV-02: 최저 σ 후보
```

### 검증 논리

- **LIT**: 시뮬레이터가 알려진 결합 친화도 순위를 재현하는가?
- **SAN**: 핵심 잔기(Trp8, Lys9) 치환 시 ΔG 악화가 관찰되는가?
- **NOV**: 파이프라인이 의미 있는 신규 후보를 도출하는가?

---

## 검증 결과

| ID | Class | Mean ΔG (REU) | dΔG vs WT | 문헌 일치 |
|--------|-----------|:-------:|:---------:|:-----:|
| LIT-01 | WT SST-14 | **-43.78** | — | — |
| LIT-02 | Octreotide | -42.11 | +1.67 | **Y** |
| LIT-03 | CST-14 | -37.30 | +6.47 | **N*** |
| SAN-01 | W8A mutant | -38.22 | +5.56 | **Y** |
| SAN-02 | K9A mutant | -39.53 | +4.25 | **Y** |
| NOV-01 | Pipeline | **-43.92** | **-0.15** | — |
| NOV-02 | Pipeline | -41.47 | +2.31 | — |

### 주요 해석

- **LIT-02 (Octreotide)**: WT 대비 소폭 약화 → 문헌과 일치 ✓
- **SAN-01/02**: 핵심 잔기 치환 시 결합 약화 확인 → Sanity check 통과 ✓
- **LIT-03**: 불일치 발생 → 별도 분석 필요 (다음 슬라이드)
- **NOV-01**: WT에 근접하는 ΔG 달성

## LIT-03 불일치 분석: 시뮬레이터 한계

### CST-14 (LIT-03)의 예상 외 결과

- **기대**: CST-14는 SST-14와 구조적으로 유사 → 유사한 ΔG 예상
- **실측**: dΔG = +6.47 (WT 대비 크게 약화)
- **문헌 일치 여부**: **N** (불일치)

### 원인 분석

```
CST-14 서열 특성
  └─ Proline 잔기 포함
       └─ 백본(backbone) 강직성 유발
            └─ FlexPepDock의 유연 리파인먼트에서 페널티
                 └─ ΔG 과대 추정 (결합 약화 방향)
```

### 시사점

| 구분 | 내용 |
|------|------|
| **한계** | FlexPepDock은 Proline-rich 펩타이드의 결합 에너지를 과소평가 |
| **영향 범위** | Proline 포함 후보에 대한 QC 판정 왜곡 가능 |
| **대응 방안** | Proline 전용 QC 게이트 보정 규칙 개발 필요 |

## 신규 후보 성과: NOV-01 & NOV-02

### NOV-01: WT 근접 결합 달성

| 항목 | 값 |
|------|-----|
| Mean ΔG | **-43.92 REU** |
| dΔG vs WT | **-0.15** (WT보다 미세하게 우수) |
| 돌연변이 수 | 3개 (비핫스팟 위치) |
| 핵심 모티프 | **Trp8-Lys9 결합 모티프 보존** |

> Pharmacophore를 보존하면서 비핫스팟 위치만 변이 → WT 수준 결합 유지

### NOV-02: 최고 재현성

| 항목 | 값 |
|------|-----|
| Mean ΔG | -41.47 REU |
| dΔG vs WT | +2.31 |
| σ (표준편차) | **3.53** (전체 후보 중 최저) |

> 10회 독립 시행에서 가장 일관된 도킹 포즈 → **결합 포즈 안정성 최고**

### 종합 평가

- NOV-01: 결합 친화도 관점에서 가장 유망한 신규 후보
- NOV-02: 재현성 관점에서 가장 신뢰할 수 있는 후보

## 핵심 기여 및 의의

### 본 연구의 의의는 절대적 예측 정확도가 아닌, **워크플로우 자동화 설계**에 있음

| 기여 항목 | 내용 |
|-----------|------|
| **다중 에이전트 오케스트레이션** | 5개 전문 에이전트가 역할 분담하여 전체 루프 자동 수행 |
| **반복 간 피드백 메커니즘** | Critic → Planner 피드백으로 전략 자동 조정 |
| **추적성 확보** | Reporter 에이전트가 모든 결정과 결과를 구조화 기록 |
| **재현성 보장** | 동일 파라미터 → 동일 결과 (연구자 편차 제거) |
| **편향 완화** | DiversityManager가 돌연변이 위치 편향 자동 억제 |
| **정량적 검증 프레임워크** | LIT/SAN/NOV 3-class reference set 기반 체계적 평가 |

### 기존 대비 차별점

```
기존: 개별 도구 (PyRosetta, FoldMason, ...) → 수동 연결 → 기록 누락
본 시스템: 도구 통합 + 에이전트 오케스트레이션 + 자동 기록 → 완전 자동화 루프
```
