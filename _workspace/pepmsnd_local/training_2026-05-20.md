## 8. 결론

- **학습 상태**: 파이프라인 **완료**(데이터·전처리·학습·체크포인트·추론 스크립트·라우터·가드 등록).
- **예측 품질**: **실패에 가깝다** — test **R²≈-0.03**, Spearman **≈-0.12**, 벤치마크(SST14)도 문헌과 **크게 불일치**.
- **Layer 2 운영**: 코드 경로는 열렸으나 **P4 / HEURISTIC 취급**, Hard Cutoff에는 **현 상태로 사용 불가** 권고.

## 9. 실패 원인·우회

1. **데이터–라벨 잡음**: 단위·텍스트(`8.5 ±2.4 hours` 등)·assay 혼재 → 노이즈 큼.
2. **단순 그래프 특징**: SE3·효소 descriptor·pepMSND 원천 피처 부재.
3. **분포**: 필터로 D-AA 대부분 탈락 → **Layer 2 목적(D-AA)과 학습 분포 불일치**.
4. **DGL/libnvrtc**: 고급 그래프 스택 재현 제약.

**우회 제안**

- PEPlife2에서 **표준화된 assay subset** 만 필터 (예: species/media 단일).
- **log1p 외에** censored regression / 구간 손실 검토.
- **RDKit 외 1D sequence encoder** (CNN/Transformer) 병합.
- D-AA: **분자=graph** SMILES 생성 경로 구축 (Open Babel / Chiral SMILES) 후 소수 **확인된 D-AA** 로 fine-tune.
- 장기: **공식 분류 헤드**로 맞출 경우 **안정/불안정 이진 라벨** 정의를 PEPlife2에서 재작업.
