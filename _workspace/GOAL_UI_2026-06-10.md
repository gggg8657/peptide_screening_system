## 0. Mission (한 문장)

**SSTR2 선택성 무한 발굴 엔진(continuous.py)을 웹에서 직접 시작·정지·튜닝하고, 매 epoch 의 중간 실험 결과·LLM inference·도킹 후보 구조(Mol*)·진행 상태를 실시간 모니터링할 수 있는, 잘 모듈화된 풀스택 웹 서비스를 디자인부터 구축한다.**

핵심 동사: **디자인 → FE/BE 개발 → 모듈 연결 → 웹에서 모니터링 & 실험 시작 가능**.

## 1. 왜 (현황 + 갭 — 환각 금지, 실측 기반)

### 이미 있는 것 (재사용·확장 대상)
- **BE**: FastAPI(`backend/`, port 8787), 40+ 엔드포인트 — `/api/status`, `/api/experiment/{run,stop,status,config,models}`, `/api/agents/{run_id}/stream`(SSE), `/api/structures/{path}`(PDB), `/api/images/{path}`, `/api/selectivity/*`, `/api/analysis/*`, `/api/validation/*`. `StatusEmitter`(원자적 write → `/tmp/pipeline_local_status.json`) 풍부한 상태 모델.
- **FE**: React 19 + Vite + TS + Tailwind(OKLCH 토큰) + **Mol* 5.6.1** + React Query(2s/3s 폴링) + Recharts + Zustand. 페이지(RunConsole/Candidate/SelectivityExplorer/RunLauncher), 컴포넌트(AgentMonitor·LoopTimeline·MoleculeViewer·ConvergenceGraph·QCGateChart·CandidateTable·ADMETPanel·ClusterPanel), Vitest 테스트.

### 갭 (이 goal 이 닫을 것)
1. **무한 엔진 미연결** — `continuous.py` 의 `discovery_status.json`(epoch 진행·다양성 레벨·global best Δmargin·passing_count·history), `global_selectivity_leaderboard.json`(역대 Δmargin 리더보드), `baseline_cache.json` 가 **어떤 엔드포인트로도 노출 안 됨**, UI 도 없음.
2. **무한 엔진 제어 부재** — 웹에서 continuous 엔진 시작/정지(STOP 파일)/실시간 튜닝(`discovery_control.json`)하는 컨트롤 없음. (기존 `/api/experiment/run` 은 단발 run 전용.)
3. **LLM inference 상세 모니터링 약함** — Planner/Critic/Reporter 의 `report` 일부만 노출. 선택성 진단·Critic 사유·재시도·프롬프트/응답 추적 미흡.
4. **Mol* ↔ 후보 PDB 경로 불일치** — MoleculeViewer 가 `05_docking/pose_*.pdb` 가정. 실제 엔진 컨벤션 `iter_NN/cand_NNN.pdb` + 글로벌 리더보드 후보 PDB 와 연결 필요.
5. **라이브 진행 인디케이터 부족** — "돌아가는 무언가"(epoch 스피너, 단계 진행바, agent ping, ETA, 토큰 스트림) 통합 부재.

## 3. Phase 계획 (서브에이전트 대량 활용 — Fan-out/Fan-in + Producer-Reviewer)

> 각 Phase 끝에 **reviewer-code**(모듈화·클린코드·테스트) + 해당 도메인 리뷰어 검증 루프(≤3). 모든 산출물은 `_workspace/` 컨벤션.

### Phase 0 — 정밀 진단 & 계약 정의 (Explore ×N 병렬)
- BE 상태모델·엔드포인트, FE 컴포넌트·훅, 무한엔진 출력파일 스키마 재확정(본 문서 §1 기반).
- **FE/BE 공유 타입 계약**(TypeScript ↔ Pydantic) 정의 — `discovery_status`, `leaderboard entry`, `candidate`, `agent inference event` 스키마 단일 소스.

### Phase 1 — UX/UI 디자인 (reviewer-uiux + researcher 병렬)
- 선행 서비스 벤치마킹(researcher), 기능 인벤토리·IA·와이어프레임·컴포넌트 스펙(reviewer-uiux). §2 산출물.

### Phase 2 — 백엔드 모듈 (engineer-backend, 엔드포인트별 fan-out)
- 무한엔진 라우터 신설: `GET /api/discovery/status`(discovery_status.json), `/api/discovery/leaderboard`(global), `/api/discovery/epochs`(history), `/api/discovery/control`(GET/PUT — control 파일 안전 편집, 화이트리스트), `POST /api/discovery/start|stop`(STOP 파일·프로세스).
- **LLM inference 이벤트 스트림**: Planner/Critic/Reporter 프롬프트·응답·사유·토큰을 구조화 로깅 → SSE/`GET /api/discovery/llm-trace`.
- 후보 PDB 경로 정규화 엔드포인트(글로벌 리더보드/iter 후보 → `/api/structures` 일관 매핑).
- Pydantic 모델 = Phase 0 계약. pytest.

### Phase 3 — 프론트엔드 모듈 (컴포넌트별 fan-out — /subagent-dev 또는 Workflow)
- **무한엔진 대시보드**: epoch 타임라인, 다양성 레벨 게이지, global best Δmargin 추이, passing_count, history 차트.
- **글로벌 리더보드 뷰**: Δmargin 정렬 후보 테이블 + 각 행 → Mol* 후보 PDB 뷰 연결.
- **LLM inference 라이브 뷰**: 에이전트별 프롬프트/응답/사유 스트림(LangSmith 풍), 토큰 진행.
- **Mol* 후보 뷰어 정비**: `iter_NN/cand_NNN.pdb` + 리더보드 후보 로드, off-target/on-target 토글, 결합 포켓 하이라이트.
- **실험 제어 패널**: 무한엔진 start/stop + control 라이브 튜닝(n_candidates·max_iterations·patience·selectivity_max_per_iter) + STOP.
- **라이브 진행 인디케이터**: 글로벌 상태바(현재 epoch·단계·ETA·agent ping·스피너), "running" 비주얼.
- 각 컴포넌트 = 독립 모듈 + 타입 계약 + Vitest.

### Phase 4 — 통합 & E2E 검증
- BE↔FE 연결, 실제 무한엔진 **1~2 epoch 가동** → 웹에서 실시간 모니터(중간 결과·LLM·Mol*) + 시작/정지/튜닝 동작 검증. vLLM(8000)·bio-tools 전제.

### Phase 5 — 모듈화/리팩토링 마감 (reviewer-code 주도)
- feature-sliced 구조 정리, API 클라이언트 단일화, god-component 제거, 공유 타입 계약 확정, 테스트·문서(README·CONTINUOUS_DISCOVERY.md UI 절 추가).

## 6. 성공 기준 (측정 가능 — 실행 전 확정)
1. 웹에서 **무한 발굴 엔진 시작·정지·실시간 튜닝** 가능 (STOP·control 연동).
2. **라이브 모니터링**: epoch 진행·다양성 레벨·global best Δmargin·passing_count·중간 후보·QC 게이트 실시간 갱신.
3. **LLM inference 모니터링**: Planner/Critic/Reporter 의 프롬프트·응답·사유를 웹에서 추적.
4. **Mol* 후보 구조 뷰**: 임의 도킹 후보(iter/리더보드) PDB 를 웹에서 로드·회전·뷰 모드 전환.
5. **라이브 진행 인디케이터**: 상태바/스피너/agent ping 등 "돌아가는" 시각 피드백.
6. **모듈화**: 신규 기능이 feature-sliced 독립 모듈 + 타입 계약 + 테스트. reviewer-code 통과.
7. **E2E**: 실제 엔진 1 epoch 을 웹에서 끝까지 모니터 + 제어 1회 성공.
