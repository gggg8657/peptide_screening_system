### test_critic_normalization.py 요약

**목적**: `AG_src/agents/critic.py`의 `ScientistCriticAgent._normalize_failure_breakdown()` 및 `analyze_results()` 함수 검증

**배경 (파일 docstring)**:
> `qc_ranker._build_qc_report()`는 fail_reason 첫 어절(예: `"pLDDT_mean"`, `"ddG"`)을 `failure_breakdown` 키로 사용. `critic.py`의 `FAILURE_ACTION_MAP`은 `FailureType.*` 상수(예: `"low_plddt"`)만 인식하므로, 정규화 없이 사용하면 `propose_changes`에서 모든 키가 스킵 → `changes=[]` → 하드코드 fallback 가설 무한 반복.

**테스트 클래스 7개**:

| 클래스 | 검증 내용 |
|--------|---------|
| `TestNormalizeFailureBreakdownDirect` | `pLDDT_mean→LOW_PLDDT`, `ddG→GOOD_DOCK_BAD_DDG` 등 직접 매핑 6건 |
| `TestNormalizeAccumulation` | 같은 FailureType으로 여러 키 누적 합산 |
| `TestNormalizeUnknownSkip` | 매핑 불가 키 조용히 스킵 |
| `TestNormalizeFuzzyFallback` | `"SomePLDDT_value"` 등 부분 일치 퍼지 매핑 6건 |
| `TestNormalizePassthrough` | 이미 정규화된 FailureType 키 그대로 통과 |
| `TestAnalyzeResultsIntegration` | `analyze_results()` 엔드투엔드 — `proposed_changes` 비어있지 않음 검증 |
| `TestGenerateHypothesisRegression` | `pass_rate=0.0` + 빈 breakdown → 동적 메시지 / `pass_rate≥0.1` → 하드코드 fallback |

**실행 흔적**: `__pycache__` 내 `.pyc` 존재 → 이미 1회 이상 실행됨. pytest 9.0.2 환경.

**커밋 미완**: `git status`에 untracked — 아직 `git add` 되지 않음. 향후 커밋 필요.
