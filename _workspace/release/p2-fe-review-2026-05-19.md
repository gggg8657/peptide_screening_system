## 7. Medium 이슈

### M-FE-1. `parseInt(raw, 10)` 부분 파싱 — 잔기 ID 입력
**[MEDIUM — 신뢰: HIGH]**  
**파일**: `BindingPocketEditor.tsx:251`

```typescript
const id = parseInt(raw, 10)
// parseInt("208abc", 10) → 208 (허용됨 — 의도치 않은 값 저장 가능)
```

`"208abc"` → `208` 로 처리됨. 사용자가 오타(또는 의도적으로 잘못된 값)를 입력해도 조용히 추가됨.

**권장 수정**:
```typescript
const id = Number.isInteger(+raw) && +raw > 0 ? +raw : NaN
if (isNaN(id)) return
```
또는 정규식: `/^\d+$/.test(raw.trim())` 사전 검증.

## 8. Low 이슈

### L-FE-1. 잔기 ID 최대값 제한 없음
**[LOW — 신뢰: HIGH]**  
`parseInt("99999999", 10)` → 99999999 추가됨. 실제 SSTR2 잔기는 1~350 범위. `id > 5000` 경고 또는 거부 권장.

### L-FE-2. 좌표 NaN 처리 `parseFloat(...) || 0`
**[LOW — 신뢰: HIGH]**  
`BindingPocketEditor.tsx:467`: 잘못된 문자열 입력 시 `NaN || 0 = 0` 으로 묵시적 리셋. 사용자가 `-` 입력 후 숫자를 추가하기 전에 좌표가 0으로 날아가는 UX 문제. controlled input 패턴으로 개선 권장 (입력 중 string 유지, blur 시 변환).

### L-FE-3. 탭 키보드 화살표 내비게이션 미구현
**[LOW — 신뢰: HIGH]**  
ARIA tabs 패턴(WAI-ARIA 3.22)은 좌/우 화살표로 탭 포커스 이동을 권장. 현재 Tab 키로만 이동 가능. 비차단.
